# 模型同步引擎 AI 文档增强版手工测试用例

## 测试目标

验证“模型自动同步引擎 AI 文档增强重构”在控制台与开发者侧的手工可见行为，重点确认：

- 自动同步与手动同步是否按两层架构工作
- 模型、价格、上下文窗口在页面上是否正确展示
- AI 补全失败时页面是否优雅降级
- `sellPriceLocked` 是否保护手工卖价
- 管理端与开发者侧是否正确反映最终同步结果

## 测试环境

- 环境类型：本地 Codex 测试环境
- 推荐初始化方式：`bash scripts/test/codex-setup.sh`
- 访问地址：`http://localhost:3099`
- 说明：本文件仅定义手工测试用例，当前未执行

## 测试范围

- 管理端登录与同步入口
- 管理端 Models & channels 页面
- 管理端价格编辑与锁价行为
- 开发者模型列表展示
- 同步日志展示
- AI enrichment 降级表现

## 前置条件

- 已部署包含本需求实现的最新代码
- 本地测试环境已启动在 `3099`
- 管理员账号可用
- 开发者账号 / API Key 可用
- 如需验证成功路径，尽量准备可用 Provider 凭证和内部 AI 通道

## 手工测试用例

### TC-MAN-001 启动应用后自动同步正常

- 优先级：P0
- 目标：验证应用启动后可自动触发同步
- 步骤：
  1. 执行 `bash scripts/test/codex-setup.sh`
  2. 打开管理端
  3. 查看同步状态或启动日志
- 预期结果：
  - 自动同步被触发
  - 可看到各 Provider 同步结果
  - 不因某个 Provider 失败导致整个启动流程中断

### TC-MAN-002 管理端可手动触发 “Sync models”

- 优先级：P0
- 目标：验证管理员可以手动触发同步
- 步骤：
  1. 登录管理端
  2. 进入模型管理相关页面
  3. 点击 `Sync models`
- 预期结果：
  - 同步任务成功触发
  - 页面出现进行中 / 完成状态
  - 最终能看到本次同步结果摘要

### TC-MAN-003 OpenAI 模型由 `/models` + AI 定价页共同补全

- 优先级：P0
- 目标：验证 OpenAI 页面结果符合两层设计
- 步骤：
  1. 同步完成后在管理端搜索 OpenAI 模型
  2. 抽查 `gpt-*`、`o3*`、`o4*`、`dall-e-3`
- 预期结果：
  - 页面只显示 chat / image 相关模型
  - 价格与上下文已补全
  - 不出现 whisper / embedding / davinci 等非目标模型

### TC-MAN-004 Anthropic 价格由文档提取补全

- 优先级：P0
- 目标：验证 Claude 系列价格可见
- 步骤：
  1. 搜索 `claude`
  2. 查看代表模型的价格与能力信息
- 预期结果：
  - 模型来自 `/models`
  - 价格已在页面展示
  - 能力字段存在时可见

### TC-MAN-005 DeepSeek 命名映射符合规范

- 优先级：P0
- 目标：验证 DeepSeek 页面命名
- 步骤：
  1. 搜索 `deepseek`
  2. 查看对应模型项
- 预期结果：
  - `deepseek-chat` 对应展示为 `deepseek/v3`
  - `deepseek-reasoner` 对应展示为 `deepseek/reasoner`
  - 价格与上下文已补全

### TC-MAN-006 智谱人民币价格换算后展示正确

- 优先级：P0
- 目标：验证 Zhipu 价格展示为美元口径
- 步骤：
  1. 搜索 `glm-4`
  2. 抽查多个模型价格
- 预期结果：
  - 页面显示美元价格
  - 价格非空时与人民币换算逻辑一致
  - 图片模型与文本模型展示正确区分

### TC-MAN-007 火山引擎模型完全来自 AI 文档提取

- 优先级：P0
- 目标：验证 Volcengine docs-only 路径
- 步骤：
  1. 搜索 `volcengine` 或豆包系列模型
  2. 统计或观察模型数量
  3. 打开某个通道详情
- 预期结果：
  - 即使没有 `/models` API，也能看到模型列表
  - 模型数量明显多于旧实现的少量静态模型
  - 价格已展示
  - 若 endpoint 未回填，界面可观察到待运营补充的痕迹或同名 `realModelId`

### TC-MAN-008 SiliconFlow 仅展示 chat / image 模型

- 优先级：P0
- 目标：验证 SiliconFlow 页面过滤结果
- 步骤：
  1. 搜索 `siliconflow`
  2. 抽查一批模型
- 预期结果：
  - 不出现 embedding / rerank / audio 项
  - 大量 chat / image 模型可见
  - 缺失价格的模型已尽量被补全

### TC-MAN-009 OpenRouter 不触发 AI enrichment

- 优先级：P1
- 目标：验证 OpenRouter 页面结果来自 API
- 步骤：
  1. 搜索 `openrouter`
  2. 查看同步摘要或日志展示
- 预期结果：
  - 模型数量大
  - 价格和上下文齐全
  - 同步说明体现 “AI not needed” 或等价含义

### TC-MAN-010 AI 补全失败时页面降级显示

- 优先级：P0
- 目标：验证 AI enrichment 失败不影响页面基本可用
- 步骤：
  1. 构造某 Provider 文档提取失败场景
  2. 重新同步
  3. 打开模型页
- 预期结果：
  - 页面仍可展示第 1 层同步结果
  - 缺失价格显示为 `—`
  - 不出现整页报错或白屏

### TC-MAN-011 AI 只补不覆盖已有 API 数据

- 优先级：P0
- 目标：验证合并规则在页面上的表现
- 步骤：
  1. 找一个 API 已返回上下文窗口的 Provider 模型
  2. 重新同步
  3. 对比同步前后页面字段
- 预期结果：
  - API 已有字段值不被 AI 改写
  - 仅新增缺失字段

### TC-MAN-012 `pricingOverrides` 为空时页面不报错

- 优先级：P1
- 目标：验证无运营覆盖配置时页面稳定
- 步骤：
  1. 确保某 Provider 未配置 `pricingOverrides`
  2. 完成同步
  3. 打开模型页
- 预期结果：
  - 页面正常
  - 不出现配置缺失报错

### TC-MAN-013 手工卖价锁定后再次同步不被覆盖

- 优先级：P0
- 目标：验证 `sellPriceLocked`
- 步骤：
  1. 管理端选择一个已有通道
  2. 手工修改卖价并开启锁定
  3. 再次点击 `Sync models`
  4. 刷新页面查看通道详情
- 预期结果：
  - 手工卖价保持不变
  - 锁定状态保持为开启
  - 成本价与来源统计可更新，但卖价不回退

### TC-MAN-014 控制台同步日志正确显示来源统计

- 优先级：P1
- 目标：验证同步日志可读性
- 步骤：
  1. 完成一次全量同步
  2. 查看同步日志区域
- 预期结果：
  - 每个 Provider 有独立统计
  - 能区分 API 数量、AI 补充数量、override 数量
  - OpenRouter 显示 AI 不需要

### TC-MAN-015 管理端 Models & channels 页面来源展示正确

- 优先级：P0
- 目标：验证模型聚合与 Channel 来源展示
- 步骤：
  1. 打开 `Models & channels`
  2. 搜索跨 Provider 可用模型，如 `gpt-4o`
  3. 展开通道来源
- 预期结果：
  - 同底层模型聚合为一个 Model
  - 下面可看到多个来源 Channel
  - 主命名优先为直连 Provider 规范名

### TC-MAN-016 开发者模型列表页展示正确

- 优先级：P0
- 目标：验证开发者侧模型页
- 步骤：
  1. 使用开发者账号登录
  2. 打开模型列表页或调用对应页面接口
  3. 抽查文本模型、图片模型、锁价模型
- 预期结果：
  - 模型列表可见
  - 卖价显示正确
  - 缺失成本信息的模型不会展示错误价格

### TC-MAN-017 内部 AI 调用不会污染开发者调用记录

- 优先级：P0
- 目标：验证同步内部调用不写开发者可见流水
- 步骤：
  1. 记录同步前开发者日志 / 账单 / 调用记录
  2. 触发一次需要 AI enrichment 的同步
  3. 刷新开发者侧日志 / 账单页面
- 预期结果：
  - 不出现同步任务产生的调用记录
  - 不产生开发者余额变化

### TC-MAN-018 编译与页面最终可用性检查

- 优先级：P0
- 目标：验证实现最终可交付
- 步骤：
  1. 完成一次构建验证
  2. 打开管理端与开发者端关键页面
- 预期结果：
  - TypeScript 编译通过
  - 关键页面可正常访问
  - 不出现与本需求相关的运行时报错

## 手工回归关注点

- 旧版硬编码价格是否仍残留
- Volcengine 是否仍错误依赖 `staticModels`
- OpenRouter 是否被错误触发 AI enrichment
- 同步后通道是否意外变为 `DEGRADED`
- 管理端搜索与筛选是否被新聚合逻辑影响
- `/v1/models` 与管理端聚合结果是否一致
