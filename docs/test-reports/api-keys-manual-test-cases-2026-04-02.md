# API Keys 手工测试用例

## 测试目标

基于 [api-keys-frontend-spec.md](/Users/yixingzhou/project/aigcgateway/docs/api-keys-frontend-spec.md) 与 [api-keys-backend-spec.md](/Users/yixingzhou/project/aigcgateway/docs/api-keys-backend-spec.md)，整理控制台 API Keys 页面与相关交互的待执行手工测试用例，供 Claude 开发完成后执行验收与回归。

本文件仅定义测试用例，不代表已执行。

## 测试环境

- 环境类型：本地测试环境
- 目标地址：`http://localhost:3099`
- 测试角色：
  - 已登录管理员
  - 有项目访问权限的普通成员
  - 无目标项目权限的用户

## 测试范围

- `/keys` 列表页
- Create API Key Modal
- `/keys/[id]` 单 Key 设置页
- 与后端扩展字段联动的页面状态
- 列表搜索、分页、撤销、只读展示
- 成功态、空态、错误态、权限态

## 前置条件

- 本地测试服务可正常打开 `/keys`
- 测试项目下具备 ACTIVE / REVOKED 两类 Key
- 至少有一条带 `description` / `permissions` / `expiresAt` / `rateLimit` / `ipWhitelist` 的 Key
- 浏览器可正常使用 Clipboard API
- 如需验证 `/keys/[id]`，后端 GET/PATCH 路由已开发完成
