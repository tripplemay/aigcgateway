# StablEdge (稳定边缘) 智能流程情报目录

**调研日期：** 2026-04-17
**调研对象：** `https://www.stableisland.cn/flows`（汤汤公司租户，company_id=17）
**目的：** 作为我们公共模板库选题的市场情报参考。**仅收集元数据（名称 / 变量 / 开场 / 使用引导），不复制对方的 system prompt 或 prompt 内容**。

## 一、整体观察

- **28 个流程**，均为 `chatflow` 类型（Dify 引擎）
- **24 / 28** 启用文件上传作为主要输入；允许文件类型覆盖图片（PNG/JPG/WEBP/GIF）+ 文档（PDF/DOCX/XLSX/PPTX/MD/TXT/EML 等）+ HTML
- **5 个** 流程有显式命名变量（`files` / `photos` / `url`）
- **0 个** 配置了 `suggested_questions`（提问建议）
- **10 个** 有 `how_to_use` 快捷按钮（用来让 bot 自介绍，不是 prompt）
- **pricing_id** 分布：`{2, 4, 5}` → 暗示三档计费，按调用成本分层

**核心 UX 范式：** 用户上传素材（截图 / 文档 / 聊天记录） → 按流程主题处理 → 输出文本。几乎所有流程都走这个套路。

| 流程名 | UUID（前 8 位） | 输入方式 |
|---|---|---|
| TT-朋友圈-结果反馈类内容 | 4f005252 | 文件上传 |
| TT-朋友圈-花絮类内容 | dc3028b3 | 文件上传 |
| TT-朋友圈-活动类内容 | 5be8c319 | 文件上传 |
| TT-朋友圈-文化与价值观类内容 | af2c3162 | 文件上传 |
| TT-朋友圈产品型内容 | 9172149c | 文件上传 |
| TT-朋友圈-产品力可视化（企业基建版） | 9c49a87f | 文件上传 |

**洞察：** 朋友圈营销按**内容意图**做了细分：结果反馈 / 花絮 / 活动预告 / 价值观输出 / 产品卖点 / 企业实力展示 — 说明企业在做私域时需要"不同类型的朋友圈"而非一套模板打天下。

### B. 评论区运营（3 平台）

| 流程名 | 平台 | 输入方式 |

**洞察：** 同一个"评论区回复"逻辑，按平台做了 3 个独立流程 — 不同平台调性/话术差异大，不能合一。

```

                          HTML, XLSX, XLS, DOCX, CSV, EML, MSG, PPTX, PPT, XML, EPUB]
allowed_file_types: [image, document]
allowed_file_upload_methods: [local_file]
number_limits: 5
max_size_mb_by_type: { image: 512, document: 512, video: 10240, audio: 5120 }
```

## 五、版本号规律

流程名尾部带 `-V{major}.{minor}.{patch}`，观察到：
- `V0.0.1` / `V0.0.2` / `V1.0.0` / `V1.0.1` / `V1.0.6`
- 部分流程（如 IP 视频脚本）迭代到 6 个 patch，说明 **prompt 调优是主要维护工作**，不是节点拓扑变化
- 启示：我们的模板系统应该支持**版本 + 变更日志**（我们的 Action 已有，Template 需要对齐）

## 六、我们能做什么 vs 不能做什么

**能做（合法）：**
- 基于此目录的**用例列表**原创设计同等功能的模板
- 复用"文件上传→分类处理→文本输出"的 UX 范式（这是通用设计，不是专有）
- 借鉴"按内容意图细分朋友圈"的**分类思路**（分类法不受保护）
- 跑几次样本观察其**输出风格/长度/结构**作为我们质量的参照基准

**不做：**
- 不提取、不反推、不复制他们任一节点的 system prompt
- 不批量跑流程占用他们算力
- 不把他们的 `how_to_use` / `opening_statement` 文案原样搬运（是 UI 展示文案，有版权）
